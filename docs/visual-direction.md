# Freezy Visual Direction

## Brand Thesis

Freezy should feel like calm domestic intelligence.

It is not a neon AI toy and not a generic grocery app. The product should feel like a premium home operating system that quietly understands what is in the kitchen and what needs attention next.

Core brand attributes:

- composed
- intelligent
- tactile
- warm
- trustworthy

## Positioning

The visual language should sit between:

- a high-end kitchen brand
- a modern home control surface
- an editorial food publication

Avoid:

- cyberpunk AI styling
- noisy startup gradients
- crypto aesthetics
- generic SaaS dashboards

## Art Direction

### Mood

- warm stone, oat, slate, olive, brass
- daylight and soft shadow rather than nightclub contrast
- large, quiet surfaces with strong spacing
- photography used as evidence and atmosphere

### Visual Concepts

- shelves
- trays
- pantry labels
- kitchen materials
- calm appliance interfaces

## Typography

Use typography as the main premium signal.

### Roles

- Display: elegant serif for headers and hero moments
- UI Sans: modern humanist sans for labels and controls
- Mono: reserved for technical metadata only

### Recommended Web Fonts

- Display: `Cormorant Garamond`
- Sans: `Inter` or `Instrument Sans`
- Mono: `IBM Plex Mono`

### Rules

- do not use more than 1 display face, 1 sans, and 1 mono
- do not use all caps for large sections of copy
- do not use thin weights for utility UI

## Color System

### Core Palette

- Canvas: warm ivory / parchment
- Ink: deep graphite
- Surface: white with warm tint
- Accent: muted olive
- Accent Secondary: brass / amber
- Alert: terracotta

### Product States

- Fresh: sage
- Use Soon: amber
- Use First: terracotta
- Neutral Metadata: slate-gray

### Rules

- accent colors should be sparse and meaningful
- avoid bright pure black and pure white where possible
- status colors must be soft but legible

## Layout Principles

### Hierarchy

- one dominant action per screen
- one primary narrative per viewport
- cards should feel curated, not stacked blindly

### Density

- moderate density with generous breathing room
- avoid oversized padding everywhere
- information-rich surfaces should still feel calm

### Rhythm

- use consistent vertical spacing
- alternate between broad editorial sections and tighter utility lists

## Motion Principles

- motion should explain change of state
- avoid decorative bounce
- use fade, lift, reveal, and gentle scale

Preferred motion moments:

- page sections reveal in stagger
- scan overlay enters with immersive transition
- inventory cards settle in, not pop

## Trust Signals

Trust must be visible in the interface.

Expose:

- last scan time
- source of each item
- confidence where relevant
- estimated freshness language
- reviewed vs unreviewed state

Do not overclaim certainty.

## Screen Intent

### Home

Should feel like a household briefing.

Key blocks:

- daily summary
- use soon queue
- cook tonight
- recent scan
- shopping list

### Inventory

Should feel editorial and organized.

- clear category grouping
- strong item naming
- restrained metadata
- freshness encoded elegantly

### Scan Review

Should feel immersive and precise.

- large evidence image
- highlighted findings
- confidence-aware review UI
- explicit accept, edit, reject actions

### Shopping

Should feel concierge-like.

- grouped by urgency and purpose
- tied to household needs
- not just a checkbox dump

## Anti-Patterns

Do not introduce:

- glassmorphism overload
- purple gradients by default
- excessive border noise
- emoji as the only information architecture
- random icon sets
- too many chip colors
- generic AI sparkles

## Premium Checklist

Before shipping a screen, verify:

- it has a clear primary action
- typography does most of the work
- status language is careful and believable
- colors are restrained
- spacing feels disciplined
- surfaces feel substantial
- trust metadata is present where needed
